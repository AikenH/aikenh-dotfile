dotsetup · Architecture & Logic

v0.2.0 · Go + Bubble Tea · 2026-05

01 · System Architecture 系统分层

CLI / TUI → Core → Executor → System

L1 CLI / TUI cmd/dotsetup · internal/tui (Bubble Tea)
L2 Core 业务层 module · linker · deps · state · installer · profile
L3 Executor 执行层 runner.go · shell out · proxy inject · timeout
L4 System 系统层 apt / brew / curl / symlink / fs ops
数据流：modules/*.yaml → Core 解析 → State 读写 (~/.local/share/dotsetup/) → Executor 执行 → 系统变更

03 · Scenarios 六大场景执行流程

从初始化到回滚的完整生命周期

A. 初始化 bootstrap → profile → install → link DONE
B. 增量安装 TUI select → deps resolve → install DONE
C. 更新配置 git pull → symlink 自动生效 DONE
D. 冲突处理 detect conflict → backup → link DONE
E. 反向同步 sync-back (file + dir) → git diff DONE
F. 删除/回滚 unlink → restore backup → rm state DONE
CLI 命令集： status · link <mod> · rollback <mod> · sync-back · --dry-run
